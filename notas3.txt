Quando você deseja usar um repositório remoto em vez de manter todo o seu trabalho local?
Quando eu preciso trabalhar de diferentes locais, por exemplo, ou preciso trabalhar em equipe, ou até mesmo quando quero trabalhar e um projeto longo que quero manter backup

Por que você prefere efetuar pull de alterações manualmente do que deixar o Git atualizar-se automaticamente com o seu repositório remoto?
Pq nem sempre quero mandar na hora que faço o commir pata o Git, as vezes quero esperar para poder mandar.

Descreva as diferenças entre forks, clones e branches. Quando você gostaria use um em vez de outro?
forks, ou bifurcações, é uma maneira de eu copiar um projeto que já está no GitHub. Clone é para eu clonar um repositório do GitHub e branches é uma divisão no meu projeto, como uma linha do tempo alternativa.

Qual é o benefício de ter uma cópia do último estado conhecido do remote armazenado localmente?
Remete é a "caixinha" com as informações do meu reposítório, então ele é exencial para eu sempre me manter atualizado.

Como você colaboraria sem usar Git ou GitHub? O que seria mais fácil e mais difícil?
Sem o GitHub seria dífícil colaborar em projetos de sistemas. Teria que ter uma rede menor e seria bem mais difícil alcançar mais pessoa para colaborar.
Talvez a comunicação seria mais fácil, mas isso depende de muitas coisas. O mais difícil seria manter sempre a última versão, com certeza.
